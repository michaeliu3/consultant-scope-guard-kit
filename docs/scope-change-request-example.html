A practical scope change request example

When a client asks for work beyond the current project boundary, write down the requested outcome, its effect, and who decided—before changed work begins. Here is a fictional example.

CR-002: Add a case-study page

Project: Five-page marketing website refresh
Requested by: Client project lead
Date: July 8, 2026

Requested change

Add a sixth page presenting one customer case study, using client-supplied copy and images.

Desired outcome

Publish the approved case study in the same responsive layout and navigation as the original five pages.

Impact proposal

  • Deliverables: one additional page and one navigation link
  • Schedule: review two working days after final assets arrive
  • Fee: separately agreed before work begins
  • Dependency: client supplies final copy, images, and publication approval

Decision

Approved as written by the named project lead on July 9, 2026. The consultant records the decision and updates the weekly status.

What makes the record useful?

  • It describes an observable result, not just “add a page.”
  • It identifies schedule, fee, and client-input effects without pretending they are already agreed.
  • It names the decision owner and date.
  • It connects the change to the ongoing project record.

This example is a general project-operations aid, not legal advice or a substitute for a reviewed agreement.
